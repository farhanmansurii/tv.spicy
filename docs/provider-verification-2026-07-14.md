## Notes and conclusions

- The local network initially could not resolve these domains; the table reflects the approved live-network probe, not the sandbox’s initial DNS behavior.
- A 200 HTML response is only an availability signal for the configured route. It cannot independently establish that an actual video source is available, that the source permits cross-origin playback, or that a browser can play it.
- The clearest configuration defect found is CineSrc TV routing: the application builds `/embed/tv/{id}/{season}/{episode}`, while CineSrc’s own docs require query parameters (`?s={season}&e={episode}` or aliases).
- AutoEmbed and Rive Stream were documented by their first-party pages, but both the configured endpoint host(s) and their root domains failed DNS resolution during this verification, so their live status is unconfirmed beyond DNS failure.
